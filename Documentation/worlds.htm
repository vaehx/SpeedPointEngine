Worlds

SpeedPoint stores worlds in the .spw file format, which is implemented in the GameFacilities/FileSPW module.
To load a world: 

// This call will automatically clear all existing objects in the world
engine->LoadWorld("F:\\CallOfDoody\\Workspace\\terminal.spw");

Workspaces

Handosn works with Workspaces, which can be seen as a directory scope of a project.
All pathes to resources/assets will be relative to the workspace root.

World files MUST be stored directly inside the workspace root directory
This is because SpeedPointEngine::LoadWorld(const string& file) determines the root directory for resources by finding the last occurence of a backslash (\) in the given world path.
