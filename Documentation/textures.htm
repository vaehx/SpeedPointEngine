Textures

Texture management

Textures are managed in an object pool. Each texture object is identified by its specification.

When you call ResourcePool::GetTexture(specification) it will always return a valid pointer to an existing object with the given specification.
If there was no texture object with the given specification yet, the resource pool will attempt to load the texture from disk. This may not happen immediately, but the texture will be streamed in a separate thread.
You cannot modify textures retrieved by GetTexture().

To create a new empty Texture that can be modified, you can use ResourcePool::AddTexture()
